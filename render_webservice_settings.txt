Render Web Service settings

Build Command:
pip install -r requirements.txt

Start Command:
gunicorn app:app

Environment Variables:
BOT_TOKEN=your_telegram_bot_token
ADMIN_ID=your_telegram_user_id
PUBLIC_URL=https://your-render-service-name.onrender.com
CRON_SECRET=make_any_secret_word

Optional:
SUPABASE_URL=...
SUPABASE_SERVICE_ROLE_KEY=...
MPESA_ENV=live
MPESA_CONSUMER_KEY=...
MPESA_CONSUMER_SECRET=...
MPESA_SHORTCODE=...
MPESA_PASSKEY=...
MPESA_CALLBACK_URL=https://your-render-service-name.onrender.com/mpesa_callback

After deploy, open once:
https://your-render-service-name.onrender.com/set_webhook

For auto news checking, use UptimeRobot/cron-job.org every 5 minutes:
https://your-render-service-name.onrender.com/check_news?secret=make_any_secret_word

MT5 update included:
- BUY signals queue BUY_LIMIT at entry_low
- SELL signals queue SELL_LIMIT at entry_high
- Orders expire after 2 hours if not placed
- /mt5_orders converts old MARKET_* pending queue items to LIMIT as safety fallback


Premium update included:
- When user taps Get VIP Signals, bot sends scanning notification.
- Scans crypto, forex, metals and stocks.
- Confidence grading: 79-84% = B SIGNAL, 85-89% = A SIGNAL, 90%+ = A+ SIGNAL.
- Metals included: XAU/USD Gold, XAG/USD Silver, XPT/USD Platinum, XPD/USD Palladium.
- Optional env: PREMIUM_METALS_SCAN_LIMIT=4
